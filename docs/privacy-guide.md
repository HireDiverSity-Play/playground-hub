# 개인정보 처리 가이드

이 문서는 실험 서비스에서 개인정보를 어떻게 다루어야 하는지 설명합니다. 비개발자도 알아야 하는 내용입니다.

## 기본 원칙

1. **꼭 필요한 정보만 수집**합니다.
2. **수집 사실을 사용자에게 명확히 알립니다**.
3. **외부 서비스(예: OpenAI)에 사용자 입력이 전송되는지 표시**합니다.
4. **개인정보를 로그에 남기지 않습니다**.
5. **수집한 정보는 정해진 보관 기간이 끝나면 파기**합니다.

## 개인정보로 간주되는 것

- 이름, 이메일, 전화번호
- 주소, 위치 정보
- IP 주소 (로그)
- 식별 가능한 사진 / 영상
- 사용자가 입력한 자유 텍스트 (자기소개, 사연 등 — 식별 가능성이 있을 수 있음)

## 자주 하는 실수

| 실수 | 올바른 방법 |
| --- | --- |
| 사용자 입력을 그대로 로그로 남김 | 로그에 입력값 미포함 또는 마스킹 |
| AI 사용 시 외부 전송 사실 미고지 | `ai-notice.md` 또는 화면에 명시 |
| 동의 없이 이메일 수집 | 수집 항목 / 목적 / 보관 기간 표시 후 동의 받기 |
| 개인정보 처리방침 페이지 없음 | `privacy.md` 템플릿 활용해 작성 |
| 키나 DB 접속 정보를 코드 안에 적어둠 | 환경 변수로 분리, .env는 GitHub 미업로드 |

## AI 서비스에서 추가로 확인할 것

- 사용자 입력이 어느 외부 사업자(OpenAI, Anthropic 등)로 전송되는지 **사용자에게 표시**
- 민감 정보(주민번호, 회사 기밀 등)를 입력하지 말라는 안내 노출
- 입력 / 응답 원문을 자체 DB에 남길 필요가 정말 있는지 재검토 (대부분 불필요)

## DB 사용 시 추가로 확인할 것

- 접근 권한이 최소화되어 있는가
- (Supabase) 행 단위 보안(RLS)이 활성화되어 있는가
- 백업 / 덤프에 개인정보가 그대로 들어가지 않는가

## 고위험 서비스 예시

다음 유형의 서비스는 **고위험 서비스**로 분류합니다. 일반 체크리스트만으로는 부족하니 추가 검토가 필요합니다.

### 분류 기준

| 유형 | 위험 요소 |
| --- | --- |
| AI 상담 서비스 | 사용자 자유 텍스트가 외부 AI로 전송됨 |
| 연애 / 심리 / 건강 / 법률 관련 조언 서비스 | 민감 정보 입력 가능성이 매우 높음 |
| 커뮤니티 / 게시글 / 댓글 서비스 | 타인의 개인정보가 본문에 노출될 위험 |
| 파일 업로드 서비스 | 사진 / 문서에 의도치 않은 개인정보 포함 가능 |
| 위치 기반 서비스 | 사용자 위치 자체가 개인정보 |

### 사례: LoveClinic

**LoveClinic**은 사용자가 민감한 연애 / 관계 고민을 자유 텍스트로 입력할 수 있는 서비스입니다.
입력 자체에 식별 가능한 인물 정보, 직장 정보, 가족 관계 등이 포함될 가능성이 높아 **고위험 서비스로 분류**합니다.

### 고위험 서비스 추가 점검

일반 체크리스트에 더해 다음 항목을 추가로 확인합니다.

- [ ] 입력 화면에 "민감 정보(실명, 회사명, 연락처 등) 입력 금지" 안내가 있다
- [ ] 입력 원문이 자체 DB에 저장되는지 사용자에게 명확히 표시했다
- [ ] 외부 AI 사용 시 어떤 사업자로 전송되는지 표시했다
- [ ] 운영자가 입력 내용을 직접 열람할 가능성을 사용자에게 고지했다 (해당 시)
- [ ] 신고 / 삭제 요청을 받을 창구가 화면에 노출되어 있다
- [ ] 미성년자 사용 가능성에 대한 정책이 있다

## 노출 전 필수 확인

[`checklists/privacy-checklist.md`](../checklists/privacy-checklist.md)를 통과해야 메인서비스 노출 신청이 가능합니다.
고위험 서비스에 해당한다면 **위 추가 점검 항목까지 모두 통과**해야 합니다.
