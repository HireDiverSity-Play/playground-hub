# QA 가이드

> **TL;DR** — 만든 사람 외 **한 명 이상**이 모바일 + PC에서 끝까지 동작 확인. 흰 화면 / 자리표시자 / 깨진 링크 / 처리방침 미연결 없는지 점검.

QA(품질 검수)는 사용자에게 노출되기 전에 서비스가 정상 동작하는지 확인하는 단계입니다.

## QA가 필요한 이유

- 배포는 됐지만 동작이 망가져 있을 수 있다
- 모바일에서만 깨지는 경우가 많다
- 외부 API(예: AI)가 응답을 바꾸면 결과가 이상해질 수 있다
- 개인정보 노출 같은 사고는 노출 후 발견하면 너무 늦다

## QA 단계

### 1. 기능 점검
- 메인 흐름이 끝까지 동작하는가
- 에러 메시지가 사용자에게 잘 보이는가 (흰 화면이 뜨지 않는지)
- 빈 입력, 너무 긴 입력, 특수문자에서 깨지지 않는가

### 2. 화면 점검
- 모바일 / PC에서 모두 보이는가
- 버튼이 잘리거나 가려진 곳은 없는가
- 폰트, 이미지가 깨지지 않는가

### 3. 성능 점검
- 첫 화면이 약 5초 안에 뜨는가
- 외부 호출(AI 등) 시 로딩 표시가 있는가

### 4. 개인정보 점검
- [`checklists/privacy-checklist.md`](../checklists/privacy-checklist.md) 통과
- 개인정보 처리방침 페이지가 연결되어 있는가
- AI 서비스라면 외부 전송 안내 문구가 있는가

### 5. 노출 점검
- [`checklists/exposure-checklist.md`](../checklists/exposure-checklist.md) 통과

## 누가 QA하는가

가능하면 **만든 사람과 다른 한 명 이상**이 점검합니다.
혼자서 만들고 혼자서 QA하면 같은 가정에서 같은 실수를 놓칩니다.

## 환경

- 모바일 브라우저 (가능하면 iOS Safari + Android Chrome 둘 다)
- PC 브라우저 (Chrome 기준 + 가능하면 Safari/Firefox 한 번 더)
- 새 시크릿 창에서도 동작 확인 (캐시/로그인 상태로 가려진 버그 발견용)
