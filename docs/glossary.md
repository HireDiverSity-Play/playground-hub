# 용어집

이 저장소의 가이드와 체크리스트에 자주 나오는 용어를 한 줄로 풀어 적었습니다. 어려운 단어가 나오면 여기를 먼저 보세요.

> 새 용어를 만나면 PR로 한 줄 추가해주세요. 짧을수록 좋습니다.

## 배포 / 호스팅

- **배포 (Deploy)** — 만든 웹사이트를 외부에서 접속할 수 있도록 인터넷에 올리는 일.
- **Netlify** — 만든 웹사이트를 무료로 인터넷에 올려주는 외부 서비스(호스팅). 신규 실험 서비스에 우선 사용합니다.
- **Vercel** — Netlify와 비슷한 호스팅 서비스. Next.js를 만든 회사가 운영합니다. 기존부터 Vercel로 운영 중이면 그대로 둡니다.
- **호스팅 (Hosting)** — 우리 사이트의 코드를 24시간 켜진 외부 컴퓨터에서 돌려 주는 일. Netlify / Vercel이 대신해 줍니다.
- **도메인 (Domain)** — 사용자가 주소창에 치는 이름 (예: `example.com`). 도메인을 호스팅에 연결해야 그 주소로 사이트에 접속됩니다.
- **HTTPS** — 사이트 주소 옆에 자물쇠 표시가 뜨는 안전 통신. 모든 실험 서비스는 HTTPS로만 동작해야 합니다.
- **OG 이미지 / OG 메타데이터** — 카카오톡, Slack 등에 링크를 붙였을 때 미리보기로 뜨는 이미지·제목·설명. 결과 공유가 중요한 서비스(예: 퀴즈)는 이게 깨지지 않게 해야 합니다.

## 환경 변수 / 키

- **환경 변수 (env)** — 키나 비밀번호처럼 코드 안에 직접 적으면 안 되는 값을 따로 빼서 관리하는 곳. Netlify / Vercel 대시보드에 등록합니다.
- **`.env`** — 환경 변수의 실제 값을 담은 파일. **GitHub에 절대 올리지 않습니다.**
- **`.env.example`** — 어떤 환경 변수가 필요한지만 적어둔 샘플. GitHub에 올리는 용도.
- **API 키 / 시크릿 (Secret)** — 외부 서비스(예: OpenAI)를 호출하기 위한 비밀번호 같은 값. 코드나 브라우저(클라이언트)에 두면 안 되고, 서버 환경 변수로만 관리합니다.

## 데이터베이스 (DB)

- **DB (데이터베이스)** — 사용자가 입력한 정보를 저장해 두는 창고. 저장이 필요 없으면 안 써도 됩니다.
- **PostgreSQL** — 가장 널리 쓰이는 관계형 DB. Neon / Supabase 모두 이 위에서 동작합니다.
- **Neon** — 단순 데이터 저장이 필요할 때 쓰는 PostgreSQL 기반 DB. 무료 플랜이 넉넉하고 사용량이 적으면 자동으로 잠들어 비용이 거의 들지 않습니다.
- **Supabase** — DB + 로그인 + 파일 업로드를 한 번에 묶어 제공하는 서비스. 사용자별 데이터를 다루거나 로그인이 필요할 때 사용합니다.
- **스키마 (Schema)** — DB 안의 폴더 같은 개념. 한 DB에 여러 서비스가 들어갈 때 서로 섞이지 않게 폴더로 나눠 둡니다.
- **RLS (Row Level Security)** — 한 DB 안에 여러 사용자 데이터가 섞여 있을 때, **본인 데이터만 본인이 볼 수 있게 잠그는 기능**. Supabase 사용 시 사실상 필수.
- **익명 가입 (Anonymous Sign-In)** — 회원가입 화면 없이 첫 진입에 자동으로 임시 계정이 발급되는 방식. Supabase가 지원합니다.

## 개인정보 / 보안

- **개인정보** — 이름, 이메일, 전화번호, 주소, IP, 식별 가능한 사진처럼 특정 개인을 알아볼 수 있는 정보. 사용자가 입력한 자유 텍스트도 식별 가능성이 있으면 개인정보로 봅니다.
- **개인정보 처리방침 (Privacy Policy)** — 어떤 정보를, 왜, 얼마나 보관하는지 사용자에게 알리는 문서. 모든 서비스에 페이지로 연결되어 있어야 합니다.
- **이용약관 (Terms of Service)** — 서비스를 어떻게 쓸 수 있고 무엇을 보장하지 않는지 적은 문서.
- **PII (Personally Identifiable Information)** — 개인을 식별할 수 있는 정보. 일상어로는 "민감 정보"에 가깝습니다.
- **암호화 (Encryption)** — 정보를 알아볼 수 없는 형태로 바꿔 저장하는 것. 키가 없으면 원본을 볼 수 없습니다.
- **마스킹 (Masking)** — 로그나 화면에 개인정보를 그대로 남기지 않고 일부를 가리는 것 (예: `홍**`, `010-12**-1234`).
- **자유 텍스트** — 사용자가 정해진 항목(이름·이메일 등)이 아닌, 자기 생각을 그대로 적는 입력란. 식별 가능 정보가 우연히 들어갈 수 있어 위험도가 높습니다.

## AI

- **AI API** — OpenAI, Anthropic 같은 회사가 제공하는 AI 호출 창구. 사용자 입력을 그쪽 서버로 보내고 응답을 받아옵니다.
- **프롬프트 (Prompt)** — AI에게 주는 지시문. 사용자 입력 + 개발자가 미리 짠 안내문이 합쳐진 형태입니다.
- **Rate Limit (호출 제한)** — 한 사용자가 일정 시간 안에 AI를 너무 많이 호출하지 못하게 막는 것. 비용 폭주 / 어뷰징 방지가 목적입니다.
- **사용량 한도 (Monthly Cap)** — 한 달에 쓸 수 있는 AI 비용 상한선. AI 사업자 대시보드에서 설정합니다.
- **외부 전송 안내** — 사용자 입력이 외부 AI 사업자 서버로 전송된다는 사실을 화면에 표시하는 것. AI 서비스에 필수.

## 개발 일반 (GitHub / PR)

- **repo (레포 / 리포지토리)** — GitHub의 코드 저장 단위. 보통 한 서비스 = 한 레포.
- **private repo** — 외부에 공개되지 않는 레포. 실험 서비스는 보통 private로 만듭니다.
- **branch (브랜치)** — 같은 레포 안에서 평행하게 작업하는 줄기. `main` 브랜치가 운영 기준이고, 변경은 별도 브랜치에서 작업한 뒤 PR로 합칩니다.
- **commit (커밋)** — 변경 사항을 저장한 묶음. 한 커밋 = 한 의미 단위로 만드는 게 좋습니다.
- **PR (Pull Request)** — 내 브랜치의 변경을 `main`에 합쳐 달라고 요청하는 것. 이때 다른 사람이 검토(리뷰)합니다.
- **squash merge** — 여러 커밋을 하나로 합쳐 `main`에 넣는 머지 방식. 히스토리가 깨끗해집니다.
- **클라이언트 / 서버** — **클라이언트**는 사용자 브라우저, **서버**는 우리가 운영하는 쪽. AI API 키 같은 비밀은 절대 클라이언트에 두면 안 됩니다.
- **CRUD** — Create / Read / Update / Delete. 데이터를 만들고 / 보고 / 바꾸고 / 지우는 기본 동작 4가지.

## 자주 보는 약어

- **QA (Quality Assurance)** — 노출 전에 동작이 정상인지 점검하는 것.
- **CTA (Call To Action)** — 사용자가 눌러주길 바라는 핵심 버튼 (예: "신청하기", "시작하기").
- **OG (Open Graph)** — 링크 공유 시 미리보기를 만들기 위한 메타데이터 표준.
- **TL;DR (Too Long; Didn't Read)** — "너무 길어서 안 읽음" 의 줄임말. 문서 맨 위의 한 줄 요약.

## playground-hub 고유 용어

- **실험 서비스** — playground 우산 아래에서 빠르게 만들고 실험하는 작은 서비스들. 메인서비스와 분리되어 있습니다.
- **메인서비스** — 사용자가 처음 들어오는 본 서비스. 이 메뉴에서 실험 서비스로 이동하는 링크가 노출됩니다.
- **serviceLinks** — 메인서비스가 어떤 실험 서비스를 메뉴에 노출할지 관리하는 코드. **노출은 항상 여기서 결정**합니다.
- **노출 (Exposure)** — 메인서비스 메뉴에 실험 서비스가 보이는 상태. 배포되어 있어도 노출은 안 될 수 있습니다.
- **고위험 서비스** — 자유 텍스트 / AI / 민감 주제 / 파일 업로드 / 위치 정보 중 **하나라도** 다루는 서비스. 일반 체크리스트 외 추가 점검 필요.
- **자리표시자 (Placeholder)** — `[서비스명]`, `[담당자 이메일]`처럼 템플릿에 비워둔 부분. 실제 값으로 바꿔야 합니다.
- **서비스 상태값** — `sandbox` / `beta` / `active` / `paused` / `archived`. 자세한 의미는 [`exposure-policy.md`](./exposure-policy.md#서비스-상태값) 참고.
